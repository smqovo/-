# Prompt：B站视频字幕爬取覆盖率测试脚本

## 背景
我正在做一篇关于B站知识区非金钱激励机制的硕士论文。我已经有一份包含约7050条B站知识区视频数据的CSV文件（或数据库），其中包含每条视频的BV号（bvid）、aid、cid等字段。

我现在需要测试：这些视频中有多少比例能成功爬取到CC字幕（包括AI自动生成字幕和UP主手动上传的字幕）。

## 任务
请帮我写一个Python脚本，完成以下工作：

### 第一步：随机抽样
从我的数据文件中**随机抽取100条视频**（需要有bvid和cid字段），作为测试样本。

### 第二步：爬取字幕
对这100条视频，逐一调用B站字幕接口，检查是否存在字幕：

- 接口地址：`https://api.bilibili.com/x/player/wbi/v2?aid={aid}&cid={cid}` 或 `https://api.bilibili.com/x/player/v2?aid={aid}&cid={cid}`
- 需要在请求头中携带有效的Cookie（我会手动提供，从浏览器F12中复制）
- 返回JSON中 `data.subtitle.subtitles` 字段如果非空，说明有字幕
- 如果有字幕，进一步下载字幕JSON文件（URL在subtitles数组中的subtitle_url字段，需要加上 `https:` 前缀），提取所有 `content` 字段拼接为完整文本

### 第三步：统计与输出
输出以下信息：
1. **覆盖率统计**：100条视频中，有多少条成功获取到字幕，覆盖率是多少
2. **字幕文本长度分布**：有字幕的视频，字幕文本的平均字数、最小字数、最大字数、中位数
3. **按年份的覆盖率**：如果数据中有视频发布时间字段，统计不同年份的字幕覆盖率（判断早期视频是否覆盖率更低）
4. **详细结果CSV**：输出一个CSV文件，每行包含：bvid, aid, cid, 发布时间, 是否有字幕(True/False), 字幕语言, 字幕字数, 字幕文本前200字（预览用）

### 技术要求
- 每次请求之间设置随机休眠（1-3秒），避免触发B站限速
- 做好异常处理（网络超时、接口报错、字幕URL无法访问等情况都要catch住，不要中断整个脚本）
- Cookie以变量形式放在脚本开头，方便我替换
- 优先获取中文字幕（zh-CN 或 zh-Hans），如果只有英文字幕也记录
- 脚本运行时在控制台打印进度（第几条/总共几条，当前bvid，是否有字幕）

### 输入数据格式
我的数据文件是CSV格式，大致字段如下（字段名可能略有不同，请在脚本开头做好字段名映射配置，方便我调整）：
```
bvid, aid, cid, title, pub_date, view, coin, like, favorite, share, reply, danmaku, duration, ...
```

### 输出示例
控制台输出示例：
```
[1/100] BV1xx411x7xx - ✅ 有字幕 (zh-CN, 3256字)
[2/100] BV1yy411y8yy - ❌ 无字幕
[3/100] BV1zz411z9zz - ✅ 有字幕 (zh-Hans, 5102字)
...
========== 测试完成 ==========
总样本：100
有字幕：82 (82.0%)
无字幕：18 (18.0%)
有字幕视频平均字数：3,847
字数中位数：3,215
字数范围：[512, 18,430]
```

## 注意事项
- 这个脚本只用于学术研究目的的数据采集
- 如果接口返回code不为0，或者subtitle字段为空/不存在，都算"无字幕"
- 有些视频可能有多条字幕轨（如同时有zh-CN和en），优先取中文的那条
